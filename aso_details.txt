Multi-App Database Design
New Core Tables:
apps — Registered apps
id | name | package_name | store (google_play|app_store)
status (active|paused) | created_at
Example:
  "NetSafe VPN"  | com.NetSafe.VPN  | google_play | active
  "MyNewApp"     | com.MyNewApp.xyz | google_play | active
app_credentials — Per-app secrets
id | app_id | credential_type (service_account_json|api_key)
value (encrypted) | updated_at
users — Admin login
id | username | email | hashed_password | role (admin|viewer)
is_active | created_at
global_config — Shared settings (Anthropic, Telegram)
key | value (encrypted) | description | updated_at
Examples:
  anthropic_api_key | sk-ant-*** | Claude API
  telegram_bot_token | ***      | Notifications

Safety Validator (Combined, Simple)
Layer A — Hard Block:
  anonymous, unhackable, 100% secure, hack-proof,
  bypass government, military-grade (no proof),
  NordVPN, ExpressVPN, Surfshark (competitor brands)

Layer B — Evidence Check:
  Any claim → check app_facts table
  Not in app_facts → FLAG for human review

Layer C — Behavior:
  Same keyword > 3x → reject
  Title > 50 chars → reject
  > 1 metadata change today → hold
  > 5 metadata changes this week → hold

Rate Limiter (Publish Protection)
python# execution.py
MAX_PUBLISH_PER_DAY = 1
MAX_PUBLISH_PER_WEEK = 5

# Before every publish:
today_count = count_publishes_today()
week_count = count_publishes_this_week()

if today_count >= MAX_PUBLISH_PER_DAY:
    → hold, notify: "Daily limit reached"
if week_count >= MAX_PUBLISH_PER_WEEK:
    → hold, notify: "Weekly limit reached"
```

## Human Simulation Rules
```
Scheduling:
  Random time between 9 AM – 10 PM
  Never same hour 2 days in a row
  2 random skip days per week (natural)

Cadence per type:
  Review replies:   same day OK
  Short desc:       max 1x per 2 weeks
  Long desc:        max 1x per 3 weeks
  Title:            max 1x per month

Delays between actions:
  5–20 min between pipeline steps
  45 min – 3 hours between publish actions
```

## Daily Pipeline Flow
```
Step 1: Fetch data (Google Play API)
Step 2: Keyword analysis
Step 3: Claude generates suggestions
Step 4: Safety validation (3 layers)
Step 5: Store suggestions with risk scores
Step 6: Auto-publish risk 0-1 (review replies only initially)
Step 7: Notify human for risk 2+
Step 8: [7 days later] Performance check
Step 9: Rollback if metrics drop
```

## Launch Sequence
```
Week 1-2:  Settings → DRY_RUN = ON
           Everything runs but nothing publishes
           Review AI suggestion quality

Week 3:    DRY_RUN = OFF
           Only review replies (risk 0-1)
           Watch Telegram alerts

Week 4+:   Add short_desc if results good
           Let auto-approve learning work
